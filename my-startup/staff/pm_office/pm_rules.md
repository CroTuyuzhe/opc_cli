# PM Rules

## Document Standards
- PRD must include: background, goals, user stories, acceptance criteria, out-of-scope
- All user-facing text must have i18n keys
- Priority levels: P0 (must-have), P1 (should-have), P2 (nice-to-have)

## Review Process
1. Draft PRD → Self-check against template
2. Submit to Boss for approval gate
3. Approved PRD → Dispatch to Dev/UI with relevant excerpts only

## Communication Rules
- Always ask before assuming requirements
- Summarize user answers back to confirm understanding
- Flag scope changes immediately
